# treeコマンドとREADME.md更新

プロジェクト構造に変更があった場合、`tree`コマンドを実行してREADME.mdのプロジェクト構造セクションを更新してください。

## ユースケース

### 1. ディレクトリ構造の大幅な変更時

**実行タイミング**: 新しいディレクトリ追加、ファイル移動、構造の再編成後

```bash
# プロジェクト全体の構造を確認（2階層、node_modules除外）
tree -L 2 -I node_modules

# 結果をREADME.mdの「プロジェクト構造」セクションに反映
```

### 2. 特定パッケージ/ディレクトリの詳細確認時

**実行タイミング**: 特定ディレクトリの変更をドキュメント化する時

```bash
# 特定ディレクトリの構造を詳細表示
tree src -I node_modules

# 必要に応じてREADME.mdまたはdoc/配下に追記
```

### 3. ユーザーへの構造説明時

**実行タイミング**: プロジェクト構造を説明する必要がある時

```bash
# 簡潔な構造表示（1階層のみ）
tree -L 1 -I node_modules
```

## 基本的な使い方

```bash
# 特定の深さまで表示
tree -L 2

# 特定のディレクトリ・ファイルを除外
tree -I "node_modules|.next|.git"

# ディレクトリのみ表示
tree -d
```
