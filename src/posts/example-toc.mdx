# 目次機能テスト記事

この記事は目次（ToC）機能のテスト用です。

## はじめに

これは「はじめに」セクションです。見出しには rehype-slug によって自動的に ID が付与されます。

### サブセクション 1

これはサブセクションです。目次に階層構造で表示されます。

## 主要な機能

このブログの主要な機能について説明します。

### Markdown 対応

Markdown で記事を書けます。

### シンタックスハイライト

コードブロックはシンタックスハイライトされます。

```javascript
function hello() {
  console.log("Hello, world!");
}
```

### 目次自動生成

rehype-slug と tocbot を組み合わせて、記事の見出しから自動的に目次を生成します。

## 使い方のヒント

使い方のヒントをいくつか紹介します。

### 見出しレベル

見出しレベルは適切に設定しましょう。`h1`は記事タイトル用に 1 つだけ使い、セクションには`h2`、サブセクションには`h3`を使うのが良いでしょう。

### 適切な見出し設計

目次を見やすくするために、適切な見出し設計を心がけましょう。

## おわりに

これで目次機能のテスト記事は終わりです。
